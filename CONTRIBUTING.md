# Contributing Guide

貢献ありがとうございます。

## Issue を作成する前に

- 既存の Issue に同じ内容がないか確認してください
- バグ報告、機能提案、タスク、ドキュメント修正など、適切なテンプレートを選んでください
- バグ報告では、可能な限り再現手順を記載してください

## Pull Request の流れ

1. Issue を作成、または既存 Issue を確認します
2. 作業用ブランチを作成します
3. 変更を行います
4. 動作確認を行います
5. Pull Request を作成します

## ブランチ名の例

```text
feature/add-issue-template
fix/login-error
docs/update-readme
chore/cleanup-config
